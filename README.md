# CSV Data Editor — Internal Tool Case Study

This repository contains a high-level case study of an internal CSV processing tool I developed and integrated into a larger internal React-based system.

The source code is not public because the tool was built for a previous employer.

## Tech Stack

- Python
- Flask
- JavaScript
- React
- PostgreSQL
- Docker
- AWS EC2
- Git

## Problem

Operational teams needed a more reliable way to process, validate, edit, and export CSV files while reducing manual work and data inconsistencies.

## Solution

I developed an internal CSV Data Editor that allowed users to handle CSV workflows through a structured web-based interface.

The tool was later integrated into an existing internal system using React components.

## Key Contributions

- Built backend logic for CSV parsing and validation
- Implemented structured editing and export workflows
- Contributed React components for integration into the internal app
- Improved internal data handling workflows
- Supported deployment and environment management

## What I Learned

This project strengthened my backend development skills and helped me understand how to build practical tools for real users and business workflows.
